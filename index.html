时间复杂度$T\left( n \right) =O\left( f\left( n \right) \right) =O\left( n^2 \right) $，其中n为问题规模（求解问题的输入量），f（n）为，O表示f（n）的数量级.

一个算法的执行时间大致等于其所有语句执行时间的总和，而语句的执行时间则为该条语句的重复执行次数和执行一次所需时间的乘积。一条语句的重复执行次数称作语句频度（Frequency Count）。

设每条语句执行一次所需的时间均是单位时间，则一个算法的执行时间可用该算法中所有语句频度之和来度量。

但对于稍微复杂一些的算法，则通常是比较困难的，即便能够给出，也可能是个非常复杂的函数。因此，为了客观地反映一个算法的执行时间，可以只用算法中的“基本语句“的执行次数来度量算法的工作量。所谓“基本语句”指的是算法中重复执行次数和算法的执行时间成正比的语句，它对算法运行时间的贡献最大。（假设每条语句执行单位时间，频度越大，执行时间越多。f（n）为最大频度语句的阶数，当n趋于无穷大时，f（n）与重复执行最多次的语句频度成正比，系数为k。）
$$
\begin{aligned}
\text{总时间}&=\text{语句频度}*\text{每条语句执行时间}\
&=\text{语句频度}*\text{单位时间}\
&=\text{基本语句频度}*\text{单位时间}\
&=\text{基本语句频度}\
\end{aligned}
$$

$$
\text{总时间}=Cf\left( n \right) \text{，}f\left( n \right) \text{为基本语句频度}
\
\text{时间复杂度}T\left( n \right) =O\left( f\left( n \right) \right) \text{，时间复杂度为基本语句频度的阶数}
\
$$

14.

对于任意一棵树，节点数 = 边数 + 1。对于森林来说，每一棵树都是独立的，如果加上一条边便可以让两个不同的树组成一颗新树，即满足：节点数 = 边数 + 1。因此对于森林来说多一棵树就需要多一条边来组成树。

因此有：森林节点数 = 森林边数 + 1 + 独立树棵树 - 1
